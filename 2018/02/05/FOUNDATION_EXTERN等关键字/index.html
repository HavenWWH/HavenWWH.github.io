FOUNDATION_EXTERN,UIKIT_EXTERN,#define,static关键字

发表于 2018-02-05

extern

用 extern 声明外部全局变量，并不是定义一个变量，只会初始化一次, 该关键字告诉编译器，其声明的函数或变量可以在本模块或者其他模块中使用, 会在链接阶段生成的目标代码中找到此变量,FOUNDATION_EXTERN,UIKIT_EXTERN都是基于extern,所以FOUNDATION_EXTERN,UIKIT_EXTERN, extern用法其实都一样, 只是看是否兼容C++

FOUNDATION_EXTERN

FOUNDATION_EXTERN是Foundation框架的, 只是针对extern兼容C++

UIKIT_EXTERN

UIKIT_EXTERN是UIKit框架的,其实本质是extern, 只是对extern做了处理, 做了C++的兼容, 用法跟FOUNDATION_EXTERN一样

#define

宏define是定义一个变量，没有类型信息,只是文本替换。使用多少次就会替换多少次 所以define定义的常量在内存中有若干个拷贝,在预编译器影响速度、不能声明变量类型、不能使用const 修饰导致使用中值可以更改

extern与#define区别:

  1. 在比较字符串的时候,FOUNDATION_EXTERN, extern速度上比#define更快, FOUNDATION_EXTERN extern是直接比较指针的, isEqualToString

#define 与 const常量区别

  1. #define没有数据类型,只是简单的字符串替换，没有类型检查。而const是有对应的数据类型，编译的时候会进行判断, 可以避免一些错误
  2. #define是可变的,const修饰的变量是不可变的
  3. #define有多少地方使用，就替换多少次,在内存中就会存在若干个拷贝, 更占用内存, const修饰的常量,在程序运行过程中只有一份备份 的常量，没有了存储与读内存的操作，使得它的效率也很高
  4. #define是在编译的预处理阶段起作用，而const是在 编译、运行的时候起作用
  5. const常量可以进行调试的，define是不能进行调试的，因为在预编译阶段就已经替换掉了
  6. #define不能作为参数传给函数, const可以
  7. const常量存在于程序的数据段,是分配内存空间的 #define常量存在于程序的代码段

extern 与 static 修饰的区别

static 的变量地址是变的，只有在用到的地方才会创建, static如果在多个文件中调用, 那么就会生成多个变量, 只是这些变量都指向通一个值,例如上面低例子, 如果在三个文件中调用,那么就会自动生成三个变量，分别是_kHavenTest、_kHavenTest_101980120、 _kHavenTest_1019349c0，都指向同一个字符串@“kHavenTest”值, 第一次用到是原名加下划线前缀，只有再有用到会追加地址最为后缀区分

extern 的不管多少个地方用到，编译时只有一个下划线前缀的变量，因为 extern 的只有一个.m有这个定义。
