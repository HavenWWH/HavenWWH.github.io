Runtime消息发送和消息转发

iOS
  • Runtime
  • 原理

最重要的一句话: OC中实例对象调用方法实际叫做“消息传递”
接下来就来了解一下这个过程吧

消息传递

举例说明:

一个基本的对象的创建, 内部是怎么实现的

alloc是类方法，沿着isa找到Student元类对象，发现Student元类对象没有实现alloc方法, 继续通过Student的Superclass去寻找,找到NSObject元类，找到并执行alloc方法，并把alloc加入到NSObject元类对象的Class Cache中, 下次调用直接在Class Cache缓存列表中调用alloc方法

init是实例方法，沿着isa找到Student的类对象，发现没有实现init
Student沿着superclass找，找到NSObject类对象，执行init，并把init加入到NSObject的类对象Class Cache中, 下次调用直接在Class Cache缓存列表中调用init方法

再看一个例子:

实例方法的调用

1. 实例对象stu首先通过isa找到其类对象Student
2. 在Student类对象的cache方法缓存列表中, 找到就则直接跳转到相关的具体实现中去调用
3. 如果没有找到, 则在struct objc_method_list方法列表中查找此方法,找到就则直接跳转到相关的具体实现中去调用,并添加到cache方法缓存列表中
4. 还是找不到,那就会通过super_class指针沿着继承树向上去搜索, 如果到继承树的根部（一般为NSObject）还没有找到, 则进行消息转发
5. 如果消息转发没有做处理, 则调用NSObjec的方法doesNotRecognizeSelector:并报错unrecognized selector

类方法的调用

以上就是iOS是消息发送过程(俗称方法调用), 接下来看看消息转发的过程

消息转发

消息转发有三次机会, 如果三次机会都没有做处理, 则会crash

消息转发第一步, 动态方法解析(通过运行时动态添加方法)

例子说明:

动态添加方法函数

cls 想要添加的类对象
name 添加后的方法Selector名字
imp 具体的方法实现
types 方法参数的编码

动态方法解析说明:
如果这一步动态添加了方法, 那么消息转发到这里就结束, 不会再继续往下走, 只要实现了class_addMethod方法, 不管返回ture或者false都不会进行下一步

消息转发第二步, 备用接收者(是否设置备用对象处理此次消息)

当对象所属类没有动态添加方法后，runtime会查看是否有其他对象可以处理这个未知的selector

结果说明:
如果添加了备用接受者, 但是Person对象没有run方法, 那么程序会crash, 否则执行run方法 如果备用接收者不转发给其他对象, 则进入最后一步

第三步,消息重定向

消息重定向说明:
第一个要求返回一个方法签名，第二个方法转发具体的实现。二者相互依赖，只有返回了正确的方法签名，才会执行第二个方法

  1. 发送-methodSignatureForSelector: 返回一个消息签名（methodSignatureForSelector：），如果返回nil，消息转发结束。
  2. 如果返回了一个函数签名,将该消息的所有细节都封装到一个NSInvocation对象中(NSInvocation封装了原始的消息和消息的参数)，然后再调用（forwardInvocation：）进行处理

forwardInvocation:方法

就像一个不能识别的消息的分发中心，将这些消息转发给不同接收对象。或者它也可以象一个运输站将所有的消息都发送给同一个接收对象。它可以将一个消息翻译成另外一个消息，或者简单的”吃掉“某些消息，因此没有响应也没有错误。forwardInvocation:方法也可以对不同的消息提供同样的响应，这一切都取决于方法的具体实现。该方法所提供是将不同的对象链接到消息链的能力。

注意:可以通过重定向实现多继承, OC本身是不存在多继承的
